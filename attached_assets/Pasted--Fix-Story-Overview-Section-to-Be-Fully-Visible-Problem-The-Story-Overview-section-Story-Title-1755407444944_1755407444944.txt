# Fix Story Overview Section to Be Fully Visible

## Problem
The Story Overview section (Story Title → Central Conflict) in the red box is overflowing and not fully visible within the header area.

## Solution
Adjust the `StoryBuilder` component to make the overview section properly sized and scrollable if needed.

### Update the StoryBuilder Component

In your `story-builder.tsx` file, find the overview section and update it:

```typescript
export default function StoryBuilder(props: StoryBuilderProps) {
  // ... existing code ...

  return (
    <div
      id="story-frame"
      className="h-full w-full"
      style={{
        display: "grid",
        gridTemplateRows: `200px 1fr`, // Increased from 176px to 200px for more space
        gridTemplateColumns: "1fr",
      }}
    >
      {/* Overview - Fixed height with proper scrolling */}
      <div 
        className="border-b border-slate-200 px-4 py-3 overflow-y-auto"
        style={{
          backgroundColor: '#f8f9fa', // Light background to distinguish the section
          maxHeight: '200px'          // Ensure it doesn't exceed the grid row height
        }}
      >
        <div 
          className="text-xs leading-relaxed space-y-2" 
          style={{ 
            lineHeight: '1.3',
            fontSize: '11px'  // Slightly smaller font for better fit
          }}
        >
          {projectName && (
            <div>
              <span className="font-semibold text-slate-800">Story Title</span> — {projectName}
            </div>
          )}

          <div>
            <span className="font-semibold text-slate-800">Project Type</span> — {projectTypeDisplay}
          </div>

          {genre && (
            <div>
              <span className="font-semibold text-slate-800">Genre</span> — {genre}
              {genreDef && (
                <div className="ml-3 text-[10px] text-slate-600 leading-tight mt-1">{genreDef}</div>
              )}
            </div>
          )}

          {subGenre && (
            <div>
              <span className="font-semibold text-slate-800">Sub Genre</span> — {subGenre}
              {subGenreDef && (
                <div className="ml-3 text-[10px] text-slate-600 leading-tight mt-1">{subGenreDef}</div>
              )}
            </div>
          )}

          {theme && (
            <div>
              <span className="font-semibold text-slate-800">Theme</span> — {theme}
              {themeDef && (
                <div className="ml-3 text-[10px] text-slate-600 leading-tight mt-1">{themeDef}</div>
              )}
            </div>
          )}

          {subTheme && (
            <div>
              <span className="font-semibold text-slate-800">Sub Theme</span> — {subTheme}
              {subThemeDef && (
                <div className="ml-3 text-[10px] text-slate-600 leading-tight mt-1">{subThemeDef}</div>
              )}
            </div>
          )}

          {centralConflict && (
            <div>
              <span className="font-semibold text-slate-800">Central Conflict</span> — {centralConflict}
              {centralConflictDef && (
                <div className="ml-3 text-[10px] text-slate-600 leading-tight mt-1">{centralConflictDef}</div>
              )}
            </div>
          )}
        </div>
      </div>

      {/* Editor viewport - takes remaining space */}
      <div className="min-h-0 h-full overflow-hidden px-4 pb-3">
        <div className="h-full w-full">
          <RichEditor value={storyHtml} onChange={setStoryHtml} className="h-full w-full" />
        </div>
      </div>
    </div>
  );
}
```

## Recommended Solution:

Use **Option 1** (scrollable version) because:
- ✅ All content remains fully visible
- ✅ Definitions are readable
- ✅ Professional scrollbar for overflow
- ✅ Easy to scan and read
- ✅ Doesn't require truncation

This will make all the Story Overview content (Story Title → Central Conflict) properly visible within the red box area.
